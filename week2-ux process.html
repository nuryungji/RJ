UX 디자인 프로세스와 리서치

UX 프로세스

크게 문제를 발견하고 해결하는 단계로 나눌 수 있으며 사용자 리서치를 통해 문제를 발견하고 UX 전략 수립을 통해 나아갈 방향을 제시하며
이러한 전략을 토대로 서비스 기획과 사용자 검증을 하는 단계로 나아갑니다.

1. 리서치 단계:

사용자 연구: 설문조사, 인터뷰, 행동 관찰 등을 통한 사용자의 요구사항, 목표, 행동 패턴, 불편한 점 등을 파악
시장 및 경쟁 조사: 해당 분야의 시장 동향과 경쟁사의 제품 또는 서비스를 분석
컨텍스트 분석: 사용자가 제품 또는 서비스를 어떻게 사용하는지 이해하기 위해 사용 환경을 조사

2. 정의 단계:

문제 정의: 수집한 데이터를 기반으로 사용자의 주요 문제와 목표를 정의
유용성 목표 설정: 사용자 경험을 개선하기 위한 목표 설정

3. 아이디어 도출 단계:

브레인스토밍: 다양한 아이디어를 자유롭게 도출하고 기록
와이어프레임 및 스케치: 스케치나 와이어프레임을 통한 아이디어를 시각화 및 구체화
스토리보드: 사용자의 경험을 이야기 형태로 만들어 시나리오를 작성

4. 프로토타이핑 단계:

프로토타입 제작: 디자인 아이디어를 프로토타입으로 구현
테스트 프로토타입: 프로토타입을 사용자에게 제공하고 사용자의 피드백을 수집

5. 테스트 단계:

사용자 테스트: 사용자에게 프로토타입을 사용하도록 하고, 사용자의 행동과 피드백을 기록
발견된 문제 해결: 사용자 테스트 결과를 분석하고, 발견된 문제를 해결하는 방법 서칭

6. 구현 단계:

디자인 및 개발: 프로토타입을 기반으로 제품 또는 서비스를 디자인하고 개발
사용자 테스트 재실시: 개발한 제품 또는 서비스를 다시 사용자에게 테스트하고 추가 개선

7. 평가 단계:

사용자 피드백 반영: 출시한 후 사용자 피드백을 지속적으로 모니터링, 이를 기반으로 지속적인 서비스 업데이트 및 개선 진행


UX 리서치 단계

사용자에 대해 파악하고 문제를 식별하고 디자인 목표 설정을 위한 리서치 단계를 다음과 같이 진행됩니다.

1. 문제 정의 및 목표 설정:

사전분석을 통한 현재 제품 또는 서비스의 사용자 경험 문제점을 파악하고 목표 설정

2. 사용자 식별:

어떤 사용자 그룹이 제품 또는 서비스를 사용할 것인지 식별

3. 데이터 수집:

데이터 수집 방법을 선택하고 실행

4. 데이터 분석 및 문제 정의:

사용자 연구를 바탕으로 얻은 데이터를 분석하고 이를 기반으로 주요 문제와 목표를 정의 및 문서화

5. 페르소나 생성:

사용자 연구를 토대로 사용자 그룹을 대표하는 가상 인물인 페르소나 설정하고 특성, 목표, 등을 설명하는 상세 정보를 포함

6. 정보 아키텍처:

사용자의 정보를 구조화하고 어떻게 표현할 것인지를 결정하여 앱 또는 소프트웨어의 정보 구조와 내비게이션을 설계

7. 시나리오 작성:

사용자 연구를 토대로 사용자가 제품 또는 서비스를 사용하는 시나리오를 정의
